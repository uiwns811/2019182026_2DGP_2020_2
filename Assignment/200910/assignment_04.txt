gitignore은 깃에서 특정 파일이나 디렉토리를  깃의 버전 관리 대상에서 제외할 때 사용하는 설정파일이다. 파일 백업 정보 등을 공개 저장소에 올리지 않도록 하는 것이다. 또한 원치 않는 파일이 업로드 되는 것을 방지하고, 데이터 충돌이 발생하여 손실이 일어나는 경우를 예방하기 위해 사용한다. 

주로 git add에 포함시키지 않는 경우에는 보안상으로 위험성이 있는 파일, 프로젝트와 관계없는 파일, 용량이 너무 커서 제외해야하는 파일 등이 있다. 

이러한 문제를 방지하기 위해 ignore할 목록을 gitignore에 작성하여 git add 명령시 해당 파일을 제외된다. 각 라인마다 무시할 파일이나 디렉토리의 패턴을 적어주면 되며, 여러 파일을 지정할 경우 *을 사용하면 된다. 

gitignore의 대표적인 사용법은 다음과 같다.

# : 주석
빈 라인 : 아무런 영향을 주지 않는다.
*.a : 확장자가 .a인 모든 파일을 무시한다.
folder_name/ : 해당 폴더의 모든 파일을 무시한다.
folder_name/*.a : 해당 폴더의 확장자가 .a인 모든 파일을 무시한다.
folder_name/**/*.a : 해당 폴더를 포함한 모든 하위 폴더에서 확장자가 .a인 모든 파일을 무시한다.
/*.a : 현재 폴더의 확장자가 .a인 모든 파일을 무시한다.

2)
venv : venv는 가상환경 폴더로, Python 가상환경은 각 프로젝트마다 Python 모듈 및 라이브러리의 버전을 독립적으로 유지하기 위해 필요한 환경이다. 또한 운영체제 마다 가상환경 폴더의 구성이 다르므로 OS 독립성을 유지하기 위해 가상환경 폴더를 Git 소스 제어에서 제외시켜야 한다.

settings.py : 프로젝트 폴더 안에 있는 settings.py를 보면  SECRET_KEY 변수가 존재한다. 키의 값은 프로젝트가 생성될 때마다 랜덤으로 할당되는데, 이 키는 django 내에서 로그인과 같은 세션을 유지하고, 비밀번호 생성, 쿠키, 암호화 서명 등에 사용되는 키이다. 이 키 값을 공격자가 탈취하면 서비스의 암호화된 로직이나 데이터에 접근할 가능성을 크게 열어준다. 따라서 이 키를 소스 제어에 포함시키거나 커밋시키지 않아야 한다. 

PyCharm 설정 폴더 : PyCharm의 Project 탐색기에서는 안보이지만 PyCharm 프로젝트의 설정을 저장하는 .idea라는 폴더가 생성된다. 상단의 Project 모드를 Project Files 모드로 바꾸어주면 프로젝트 루트 폴더의 실제 폴더 및 파일 구조가 나타난다. 이 폴더는 모두 제외시켜도 상관없지만, 프로젝트 설정을 보존하고 싶다면 특정 파일들을 제외하고 포함시켜도 된다. 

db.sqlite3(데이터베이스) : 실제 데이터가 들어있는 데이터베이스 파일은 필요없으므로 제외시켜야 한다.
